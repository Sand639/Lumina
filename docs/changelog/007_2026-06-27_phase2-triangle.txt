■ なぜそうした（設計判断）
--------------------------------------------------------------------------------
・シェーダーは埋め込み文字列 + D3DCompile(FXC)で実行時コンパイルした。
   三角形を出す目的にはこれで十分。ロードマップの「DXC + リフレクション +
   ホットリロード」は Phase 3 の主題なので、そこでファイル化+置き換える。

・ルートシグネチャはパラメータ0(空)。
   この三角形は CBV/SRV を使わず、頂点バッファ(IA)だけで描けるため。
   入力アセンブラ入力レイアウトの使用許可フラグだけ立てた。

・カリングを無効(CULL_NONE)にした。
   頂点の巻き順(時計/反時計)に依存せず確実に表示するため。巻き順を意識した
   カリングは、実モデルを扱う段階で正す。

・深度・ブレンドは最小(深度オフ/不透明)。
   1枚の三角形には不要。深度バッファは3Dを重ねる段階(Phase 4以降)で導入する。

・DPI認識(Per-Monitor V2)を入れた。
   非対応だとOSが画面を引き伸ばし、バックバッファ解像度と表示がズレてボケる。
   グラフィックスアプリとして物理ピクセルと1:1で描くのが正しいので対応した。
   (副作用として、スクリーンショットも実ウィンドウと一致するようになった)

■ 使っている技術・用語の解説
--------------------------------------------------------------------------------
・シェーダー(HLSL) / 頂点シェーダー(VS) / ピクセルシェーダー(PS)
   VS = 各頂点を処理しクリップ空間座標を出す。今回は座標をそのまま float4 化。
   PS = 各ピクセルの色を出す。今回は補間された頂点カラーをそのまま返す。
   頂点間の色は「ラスタライザ」が自動で線形補間する(=虹グラデーション)。

・D3DCompile
   HLSLソースを実行時にコンパイルし、シェーダーバイトコード(ID3DBlob)を得るAPI。

・ルートシグネチャ(ID3D12RootSignature)
   シェーダーが受け取るリソース(定数/テクスチャ等)の“引数仕様”を定義するもの。
   今回は引数なし。D3D12SerializeRootSignature でバイナリ化して生成する。

・PSO(Pipeline State Object / ID3D12PipelineState)
   描画パイプラインの設定一式(VS/PS、入力レイアウト、ラスタライザ、ブレンド、
   深度、RTフォーマット、トポロジ種別…)を1つに固めた不変オブジェクト。
   D3D12では描画状態をこのPSOにまとめて事前生成する(状態切替が速い)。

・入力レイアウト(Input Layout)
   頂点バッファのバイト列を、シェーダー入力(POSITION/COLOR)へどう割り当てるかの定義。
   Vertex 構造体(Float3+Float4)とオフセット(0/12)・フォーマットを一致させる。

・ビューポート / シザー矩形
   ビューポート=描画する画面範囲(NDC→ピクセルへの写像)。
   シザー=この矩形外のピクセルを捨てる。両方を画面サイズに設定する。

・DrawInstanced(頂点数, 1, 0, 0)
   頂点バッファの頂点を使って描画する命令。今回は3頂点・1インスタンス。

・DPI認識(SetProcessDpiAwarenessContext / Per-Monitor V2)
   OSの拡大率に対しアプリが「自分で物理ピクセルを扱う」と宣言する設定。
