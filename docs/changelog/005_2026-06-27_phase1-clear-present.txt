■ なぜそうした（設計判断）
--------------------------------------------------------------------------------
・責務ごとにクラスを分けた(CommandQueue / SwapChain / Renderer)。
   1クラス1責務(単一責任)。後で RHI 抽象(IDevice/ISwapChain…)を抽出する際、
   この分割がそのままインターフェース境界の候補になる。

・トリプルバッファ(3枚) + frames-in-flight にした。
   CPUが次フレームを準備する間、GPUは前フレームを処理…と並行させ、待ちを減らす。
   各バッファ専用のコマンドアロケータを持ち、GPU使用中のものを Reset しない。

・フェンスでバッファ単位の完了を管理した。
   各フレームの最後に Signal してフェンス値を記録。次に同じバッファを使う前に
   その値を WaitForFenceValue で待つ。これで「表示中バッファへの上書き」を防ぐ。

・SwapEffect は FLIP_DISCARD。
   D3D12 はフリップモデル必須。BitBlt 方式は使えない。

・Present は vsync 有効(同期間隔=1)。
   ティアリング防止。これにより事実上モニタのリフレッシュ(約60Hz)に同期する
   = Phase 0 の Sleep による暫定60FPSが、本来の vsync 同期に置き換わった。

・終了時は必ず Flush(GPUアイドル待ち)してから破棄。
   GPUが使用中のリソースを解放すると検証エラー/クラッシュになるため。

・リソースバリアは今はベタ書き(D3D12_RESOURCE_STATE_*)。
   ロードマップにある ResourceState enum 抽象は、RHIインターフェース抽出時に導入予定
   (今は具象を素直に書いて、後でまとめる方針)。

■ 使っている技術・用語の解説
--------------------------------------------------------------------------------
・コマンドキュー(ID3D12CommandQueue)
   GPUへ作業を投入する窓口。記録済みのコマンドリストをここに Execute する。

・コマンドアロケータ / コマンドリスト
   リスト = 描画命令を記録する帳面。アロケータ = その命令が使うメモリ領域。
   1フレーム分を記録 → Close → キューへ Execute。次フレームは Reset して再利用。
   アロケータは「GPUがそのフレームを使い終わるまで」Reset 不可 → フレーム数ぶん持つ。

・スワップチェーン(IDXGISwapChain3)
   画面表示用のバックバッファ群を束ねる仕組み。描き終えたら Present で表に出す。
   GetCurrentBackBufferIndex() で「今から描く番」を取得する。

・バックバッファ / RTV(レンダーターゲットビュー)
   バックバッファ = 画面1枚分の描画先テクスチャ。
   RTV = それを「描画先」として扱うためのビュー(ディスクリプタ)。
   ディスクリプタヒープに枚数分並べ、CPUハンドルのアドレス計算で各RTVを指す。

・リソースバリア(ResourceBarrier / 状態遷移)
   GPUリソースの「今の用途(状態)」を切り替える宣言。
   描く前: PRESENT → RENDER_TARGET、描いた後: RENDER_TARGET → PRESENT に戻す。
   これを正しく入れないと検証エラーになる(GPUに状態を教える契約)。

・フェンス(ID3D12Fence) と frames-in-flight
   フェンス = GPUの進捗を表す単調増加カウンタ。Signal(値) を積み、CPUは
   SetEventOnCompletion + WaitForSingleObject でその値到達を待てる。
   「複数フレームを同時に処理中(in flight)」にしつつ、衝突を防ぐ同期の要。

・vsync(垂直同期) / Present(同期間隔)
   Present(1,0) でモニタのリフレッシュに合わせて表示 → 画面の裂け(ティアリング)防止。

--------------------------------------------------------------------------------
■ 1フレームの流れ(Render の中身)
--------------------------------------------------------------------------------
1. 今のバックバッファ番号を取得
2. そのバッファに前回投げた作業の完了をフェンスで待つ
3. アロケータ/コマンドリストを Reset(記録開始)
4. バリア PRESENT→RENDER_TARGET
5. RTVを設定し、指定色で ClearRenderTargetView
6. バリア RENDER_TARGET→PRESENT
7. コマンドリストを Close → キューへ Execute
8. SwapChain.Present(vsync)
9. Signal し、そのフェンス値をこのバッファ用に記録
